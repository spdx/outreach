## Absolute Beginner Guide to SPDX

## What is SPDX?
SPDX is a standard format used to describe software components and their licenses. It helps organizations understand what is included in their software.

## Why is this important?
- Helps create a Software Bill of Materials (SBOM)
- Used for security and license compliance
- Widely adopted in open source and industry

## What will you do in this guide?
You will create a simple SPDX document in a few easy steps.

## Step-by-Step Guide

### Step 1: Open SPDX Online Tools
Visit the [SPDX Online Tools](https://tools.spdx.org/) to begin creating your document.

### Step 2: Enter Project Information
Add basic details such as project name and version.

### Step 3: Add a Package
Include at least one software component in your document.

### Step 4: Export the Document
Download or view your SPDX document.

## What you just did
You created a basic SPDX document (SBOM).

## Next Steps
- Explore more SPDX tools
- Try generating SBOMs for real projects

## Alternative Method: Generate SPDX using GitHub (Easier)

You can also generate an SPDX SBOM directly from GitHub without using external tools.

### Steps:

1. Open your repository on GitHub
2. Click on the **Insights** tab
3. Click on **Dependency Graph** from the left menu
4. On the top right, click **Export SBOM**
5. Download the SPDX document

This method is quick and requires no manual input.
